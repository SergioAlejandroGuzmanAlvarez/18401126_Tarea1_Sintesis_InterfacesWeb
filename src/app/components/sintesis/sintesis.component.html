Desarrollo

¿Qué es un Sistema Control de Versiones?

Un sistema de control de versiones es un software que ayuda a gestionar y rastrear los cambios en archivos y proyectos a lo largo del tiempo. Permite a los usuarios revertir a versiones previas, comparar cambios y colaborar en equipo en un proyecto. Almacena cada versión del proyecto de manera que se pueda acceder a ellas en cualquier momento y se puedan comparar entre sí para ver las diferencias.

¿Qué es Github?

GitHub es una plataforma de control de versiones de código que permite a los desarrolladores y equipos de software mantener un seguimiento de los cambios realizados en sus proyectos de manera organizada y eficiente.

Los desarrolladores pueden crear "repositorios" en GitHub, que son proyectos individuales con su propio historial de cambios y control de versiones. Los usuarios pueden "clonar" un repositorio en su propio equipo y realizar cambios en su copia local antes de "compartir" estos cambios con el repositorio principal en GitHub. Además, GitHub permite la colaboración en tiempo real entre varios desarrolladores, lo que significa que varios usuarios pueden trabajar en el mismo proyecto simultáneamente y realizar cambios en el código. Las "ramas" de Git permiten a los usuarios crear versiones separadas de un proyecto y mezclarlas posteriormente en una sola versión.

Existen tres diferentes tipos de sistemas de control de versiones en Git que pueden ayudarte a mejorar dependiendo tus necesidades los cuales son:

  1. Sistema de Control de Versiones Locales

    2. Este método es el mas "tradicional", este sucede de manera local en la computadora de la persona que este haciendo esta gestión, normalmente el usuario es quien crea esta gestión "manual" el cual muchas veces puede estar propenso a errores, como en el cual se puede olvidar uno donde quedan los archivos.

  2. Sistema de Control de Versiones Centralizados

    3. Se puede decir que es el "estandar" que todo el mundo utiliza, en donde ya no se trabaja de manera local si no en la nube junto con otros sistemas o desarrolladores, la ventaja de utilizar un sistema centralizado ayuda a que todos trabajen bajo la misma versión y bajo la misma rama permitiendoles poder tener un mejor control con los cambios.

  3. Sistemas de Control de Versiones Distribuidos

    4. Este tipo de sistema es mas común cuando se tienen sistemas "autonomos" en el cual cada sistema contiene una propia base datos en el cual puede conectarse con el servidor principal, en este caso en vez de hacer una copia de todos los archivos que se suben se hce una replica exacta en el repositorio, la ventaja de utilizar este sistema es que lo podemos programar como un "espejo" en donde si llega a caer una "computadora o servidor" lo podamos levantar con otro sin ningún problema y todo esto con el sistema de control de versiones.
